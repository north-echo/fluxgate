# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in Fluxgate, please report it responsibly.

### Contact

- **Email:** christopherdlusk@gmail.com
- **GitHub Security Advisory:** Open a [GitHub Security Advisory](https://github.com/north-echo/fluxgate/security/advisories/new) on this repository.

### What to Include

- A description of the vulnerability.
- Steps to reproduce the issue.
- The potential impact.
- Any suggested fixes, if applicable.

### Response Timeline

- **Acknowledgment:** Within 48 hours of receipt.
- **Initial assessment:** Within 7 days.
- **Resolution target:** Within 30 days for confirmed vulnerabilities.

### Scope

This policy covers vulnerabilities in the Fluxgate codebase itself. For vulnerabilities found by Fluxgate in other repositories, see [DISCLOSURE.md](DISCLOSURE.md) for our responsible disclosure protocol.
